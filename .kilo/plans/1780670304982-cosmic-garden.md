# Plan: Presentation-as-Player-UI — CSS Overlay Architecture

**Status:** Architecture analysis + open questions  
**Date:** 2026-06-05

---

## The Proposal

`<videl-presentation>` serves double duty:

- **Inactive**: a playlist card (thumbnail/poster, title, duration hint)
- **Active**: expands to fill `<videl-player>` and becomes the full player UI — seekbar, play/pause, volume, track selectors, quality selector — all rendered in its shadow DOM from the data already present in its element subtree

The `<video>` element (which lives in `<videl-player>`'s shadow DOM) is absolutely positioned to show "through" the transparent active presentation overlay, sitting visually below the controls but above the poster background.

---

## The Core CSS Stacking Model

This is the critical structural insight that makes or breaks the approach.

### Problem statement

`<video>` lives in `<videl-player>`'s shadow DOM. The controls the presentation wants to render above it are in `<videl-presentation>`'s shadow DOM, which is slotted *into* the player's shadow DOM.

For the final rendered stack to read:

```
[bottom]  poster / inactive card
          <video>
[top]     controls overlay
```

…the active presentation needs to span layers on both sides of the `<video>`. This is impossible with a single stacking context per element.

### The Solution: Transparent Overlay

The active presentation does **not** straddle the video. Instead it sits entirely **above** it, with a transparent body. The video shows through the transparent region. Controls render in an opaque strip at the bottom.

```
videl-player   { position: relative; overflow: hidden; }
video          { position: absolute; inset: 0; z-index: 1; }
slot           { /* presentations compose here */ }

/* applied by consumer / presentation's own :host rules */
videl-presentation[videl-state="active"] {
  position: absolute;
  inset: 0;
  z-index: 2;          /* above the video */
  background: transparent;  /* video shows through the body */
}
```

Inside the active presentation's shadow DOM:
```
:host([videl-state="active"]) .controls {
  position: absolute;
  bottom: 0; left: 0; right: 0;
  background: linear-gradient(transparent, rgba(0,0,0,.7));
  /* seekbar, play/pause, volume, track selectors */
}
:host([videl-state="active"]) .poster {
  display: none;  /* hide poster when video is playing */
}
```

The poster is hidden when active (video shows instead). The controls float at the bottom of the transparent overlay.

### Player shadow DOM restructuring required

`<videl-player>` currently has **no `<slot>`** in its shadow DOM:

```ts
// current
this.#shadow.innerHTML = `<style>…</style>`;
this.#video = document.createElement('video');
this.#shadow.appendChild(this.#video);
```

Without a slot, the `<videl-presentation>` children in the player's light DOM are completely invisible (this is correct for the current single-stream `src` mode, which injects a presentation programmatically). Adding a `<slot>` exposes the presentations for rendering.

New player shadow structure:
```html
<style>
  :host { display: block; position: relative; overflow: hidden; }
  video { position: absolute; inset: 0; width: 100%; height: 100%; }
  .presentations { /* layout for inactive cards */ }
</style>
<div class="presentations">
  <slot></slot>
</div>
<video></video>
```

DOM order matters here: `<slot>` (presentations) is **before** `<video>` in the shadow DOM. With no explicit z-index they stack by DOM order — video on top. The active presentation's `z-index: 2` then lifts it above the video.

## Data Available to the Presentation for UI Rendering

The presentation already owns its subtree. No DOM movement needed.

| UI Control | Data source | Already available? |
|---|---|---|
| Seekbar segments | `<videl-period start duration>` children | ✅ yes — query `this.querySelectorAll('videl-period')` |
| Seekbar position | `currentTime` from `videlUpdate(state)` | ✅ yes — store as reactive property |
| Total duration | `media-presentation-duration` attribute | ✅ yes — own attribute |
| Audio track menu | `querySelectorAll('videl-adaptation-set[content-type="audio"]')` | ✅ yes |
| Quality menu | Active adaptation-set's `videl-representation` children | ✅ yes |
| Active quality label | `videl-representation[videl-state="active"]` | ✅ yes |
| Paused state | ❌ not yet in `PlayerState` | needs addition |
| Volume | ❌ not in `PlayerState` | needs addition |

The presentation is already a `LitElement`. `videlUpdate` could stamp `currentTime` and `paused` onto reactive properties, triggering a re-render of the controls on each tick.

## State the Presentation Needs But Doesn't Have

`PlayerState` currently:
```ts
interface PlayerState {
  currentTime:    number;
  buffered:       TimeRanges;
  bandwidth:      number;
  playbackRate:   number;
  bufferAhead:    number;
  sourceBuffered: Map<string, TimeRanges>;
}
```

Required additions:
- `paused: boolean` — for play/pause button state
- `volume: number` — for volume control rendering
- `muted: boolean` — for mute button state

These are cheap to add: `videl-player` already has the video element and exposes `paused`, `volume`, `muted` as proxy getters.

## Control Command Contract (Presentation → Player)

Controls in the presentation need to trigger player actions. Since the player owns the `<video>`, commands bubble up as events:

| User action | Event dispatched | Player handles |
|---|---|---|
| Click play/pause | `videl:ui:play-pause` | calls `#video.play()` / `.pause()` |
| Click seekbar at position `t` | `videl:ui:seek` `{ time: number }` | calls `#seekTo(t)` |
| Drag volume slider | `videl:ui:volume` `{ volume: number }` | sets `#video.volume` |
| Select audio track | `videl:ui:track-select` `{ adaptationSetEl }` | sets `videl-state="active"` on the chosen ADS |
| Select quality | `videl:ui:quality-select` `{ representationEl }` | sets a "pinned" representation on the ADS |

These events bubble from inside the presentation up through the player, which catches them at the player level. This preserves the principle that the presentation never holds a direct reference to the `<video>`.

**Note on quality selection**: The current ABR logic in `VidelAdaptationSet.videlUpdate()` selects the best representation every tick. A user-pinned quality needs a way to suppress ABR. A `pinned-id` attribute on the adaptation-set (or a `videl:ui:quality-pin` event) could tell the ADS to skip ABR selection while pinned.

---

## Inactive Presentation Layout

When no presentation is active, the player should show the playlist. Since presentations are now slotted into the player's shadow DOM through a default `<slot>`, their layout is controlled by:
- The `<div class="presentations">` wrapper in the player's shadow (flex/grid layout for the card grid)
- Consumer CSS on `videl-presentation` for card sizing, spacing, thumbnail styles

When a presentation activates:
- It gets `position: absolute; inset: 0; z-index: 2` → covers the card grid
- Other cards remain in normal flow (invisible behind the overlay)

The transition between playlist view and playback view is a natural CSS overlay — no DOM restructuring, no JS-driven layout changes.

## Problems and Constraints

### 1. Adding `<slot>` to the player is a render-model change

Currently presentations are invisible (no slot). The `src` attribute mode programmatically injects a presentation that is never meant to show as a card. With a `<slot>`:
- Single-stream `src` mode: the injected presentation would render as a card/overlay (probably fine — it can be styled to be fully transparent when not explicitly styled)
- Playlist mode: presentations now render as cards — desired

**Mitigation**: style `videl-presentation` to be `display: none` by default and require the consumer to opt into the card rendering with their own CSS. Or make the injected presentation invisible by default and documented.

### 2. `videlUpdate` is pump-throttled (default 250ms)

The presentation's controls would only update every 250ms. This is acceptable for the seekbar position but may feel laggy for play/pause button state. Play/pause state could alternatively be driven by the `play` / `pause` events already forwarded from the player (the player re-dispatches video events):

```ts
// in videl-player constructor — already exists:
this.#video.addEventListener(name, () => this.dispatchEvent(new Event(name)));
```

The presentation could listen on itself for bubbled `play`/`pause`/`volumechange` events from the player, separate from the `videlUpdate` tick.

But: these events are dispatched *from the player*, not on the player. The presentation is inside the player — it would need to listen on `this.closest('videl-player')` which is an upward reference (violates the parent-knows-children direction).

**Better path**: add `paused`/`volume`/`muted` to `PlayerState` and stamp them via `videlUpdate`. 250ms is fine for these — the play state changes are user-initiated and instantly reflected at the video level; the control just needs to catch up within a tick.

### 3. User quality selection vs. ABR

Pinning a quality requires the adaptation-set to stop its automatic ABR selection. This is a new feature (`pinned-id` attribute or user-selection state). Needs design before implementation.

### 4. The `<slot>` + `position: absolute` stacking context on Firefox / Safari

CSS stacking with slotted content and absolute positioning is well-specified, but real browser behaviour has had historic quirks with shadow DOM stacking contexts. This should be verified in all target browsers during implementation.

### 5. Accessibility

Controls inside the presentation's shadow DOM need proper ARIA attributes. When the presentation is inactive, its controls must not be focusable (`inert` attribute or `display: none`). This is achievable via `:host(:not([videl-state="active"])) .controls { display: none }`.

## Detailed Implementation

### 1. `src/player-state.ts` — add playback state

```ts
export interface PlayerState {
  currentTime:    number;
  buffered:       TimeRanges;
  bandwidth:      number;
  playbackRate:   number;
  bufferAhead:    number;
  sourceBuffered: Map<string, TimeRanges>;
  // additions:
  paused:  boolean;
  volume:  number;
  muted:   boolean;
}
```

### 2. `src/elements/videl-player.ts` — shadow DOM restructure + event wiring

**Shadow DOM** (rebuilt in constructor):

```html
<style>
  :host {
    display: inline-block;
    position: relative;
    overflow: hidden;
  }
  .presentations {
    width: 100%;
    height: 100%;
    display: flex;
    flex-wrap: wrap;
    gap: 8px;
    align-content: flex-start;
  }
  video {
    position: absolute;
    inset: 0;
    width: 100%;
    height: 100%;
    display: block;
  }
</style>
<div class="presentations"><slot></slot></div>
<video></video>
```

DOM order: `<slot>` (presentations) renders first, then `<video>` renders on top via document order (both non-positioned, no z-index, so later sibling wins). Active presentation adds `z-index: 2` to lift above the video.

**`#pumpTick()`** — extend `PlayerState` construction:

```ts
const state: PlayerState = {
  currentTime:   this.#video.currentTime,
  buffered:      this.#video.buffered,
  bandwidth:     this.#bandwidth,
  playbackRate:  Math.max(this.#video.playbackRate, 0.01),
  bufferAhead:   this.#bufferAhead,
  sourceBuffered,
  paused:        this.#video.paused,   // new
  volume:        this.#video.volume,   // new
  muted:         this.#video.muted,    // new
};
```

**`#beginLoad()`** — mark injected presentation:

```ts
const presEl = parseMpd(xml, src);
presEl.setAttribute('generated', '');   // mark as auto-generated
```

**Internal UI event handlers** (add in `connectedCallback`):

```ts
this.addEventListener('videl:ui:play-pause',  this.#onUiPlayPause  as EventListener);
this.addEventListener('videl:ui:seek',        this.#onUiSeek       as EventListener);
this.addEventListener('videl:ui:volume',      this.#onUiVolume     as EventListener);
this.addEventListener('videl:ui:mute-toggle', this.#onUiMuteToggle as EventListener);
```

```ts
#onUiPlayPause = (): void => {
  if (this.#video.paused) this.#video.play().catch(() => {});
  else this.#video.pause();
};

#onUiSeek = (e: Event): void => {
  const { time } = (e as CustomEvent).detail;
  this.#seekTo(time);
};

#onUiVolume = (e: Event): void => {
  const { volume } = (e as CustomEvent).detail;
  this.#video.volume = volume;
};

#onUiMuteToggle = (): void => {
  this.#video.muted = !this.#video.muted;
};
```

Remove these in `disconnectedCallback` (mirror the `connectedCallback` removals).

### 3. `src/elements/videl-presentation.ts` — full UI shadow DOM

**New reactive properties:**

```ts
static properties = {
  // existing ...
  currentTime: { type: Number, attribute: false },  // not reflected to attr — pump-driven
  paused:      { type: Boolean, attribute: false },
  volume:      { type: Number,  attribute: false },
  muted:       { type: Boolean, attribute: false },
};

currentTime = 0;
paused      = true;
volume      = 1;
muted       = false;
```

**`videlUpdate()`** — stamp state onto reactive properties:

```ts
videlUpdate(state: PlayerState): void {
  if (this.getAttribute('videl-state') !== 'active') return;
  this.currentTime = state.currentTime;
  this.paused      = state.paused;
  this.volume      = state.volume;
  this.muted       = state.muted;
  const active = this.#childPeriods.find(
    p => p.getAttribute('videl-state') === 'active'
  );
  if (active) (active as any).videlUpdate(state);
}
```

**`render()`** — full shadow DOM:

```ts
render() {
  const totalDur = this.mediaPresentationDuration ?? this.duration ?? 0;
  const progress = totalDur > 0 ? this.currentTime / totalDur : 0;
  const periods  = this.#childPeriods;

  return html`
    <style>
      :host { display: block; position: relative; }

      /* ── Inactive card ───────────────────────────── */
      .poster { display: block; width: 100%; height: 100%; object-fit: cover; }
      .controls { display: none; }
      .click-zone { display: none; }

      /* ── Active: transparent overlay ────────────────*/
      :host([videl-state="active"]) {
        position: absolute;
        inset: 0;
        z-index: 2;
        background: transparent;
      }
      :host([videl-state="active"]) .poster   { display: none; }
      :host([videl-state="active"]) .controls {
        display: flex;
        flex-direction: column;
        position: absolute;
        bottom: 0; left: 0; right: 0;
        padding: 8px;
        background: linear-gradient(transparent, rgba(0,0,0,0.7));
        gap: 6px;
      }
      :host([videl-state="active"]) .click-zone {
        display: block;
        position: absolute;
        inset: 0;
        bottom: 60px;  /* above controls bar */
        cursor: pointer;
        background: transparent;
      }

      /* ── Seekbar ─────────────────────────────────── */
      .seek-row { position: relative; height: 20px; }
      .seek-input {
        width: 100%;
        position: relative;
        z-index: 2;
        appearance: none;
        background: transparent;
        cursor: pointer;
      }
      .period-markers {
        position: absolute;
        inset: 0;
        z-index: 1;
        pointer-events: none;
      }
      .period-mark {
        position: absolute;
        top: 50%;
        transform: translateY(-50%);
        width: 2px;
        height: 8px;
        background: rgba(255,255,255,0.5);
      }

      /* ── Control row ─────────────────────────────── */
      .ctrl-row {
        display: flex;
        align-items: center;
        gap: 8px;
        color: white;
        font-size: 12px;
      }
      .ctrl-btn {
        background: none;
        border: none;
        color: white;
        cursor: pointer;
        font-size: 16px;
        padding: 4px;
      }
      .time-display { font-variant-numeric: tabular-nums; }

      /* ── Hide technical children ─────────────────── */
      ::slotted(videl-period) { display: none; }
    </style>

    <!-- User content (poster image, title, etc.) -->
    <slot></slot>

    <!-- Inactive-only poster fallback if no user content provides one -->

    <!-- Click zone for play/pause (active only, covers video area above controls) -->
    <div class="click-zone" @click=${this.#onClickZone}></div>

    <!-- Controls (active only) -->
    <div class="controls">

      <!-- Seekbar row -->
      <div class="seek-row">
        <!-- Period boundary markers (decorative) -->
        <div class="period-markers">
          ${periods.map(p => {
            const start = Number((p as any).start ?? 0);
            const pct   = totalDur > 0 ? (start / totalDur) * 100 : 0;
            return pct > 0 && pct < 100
              ? html`<div class="period-mark" style="left:${pct}%"></div>`
              : nothing;
          })}
        </div>
        <!-- Actual seek input -->
        <input
          type="range"
          class="seek-input"
          min="0" max="1" step="0.0001"
          .value=${String(progress)}
          @input=${this.#onSeekInput}
        />
      </div>

      <!-- Button row -->
      <div class="ctrl-row">
        <button class="ctrl-btn" @click=${this.#onPlayPause}>
          ${this.paused ? '▶' : '⏸'}
        </button>
        <span class="time-display">
          ${this.#formatTime(this.currentTime)} / ${this.#formatTime(totalDur)}
        </span>
        <button class="ctrl-btn" @click=${this.#onToggleMute}>
          ${this.muted || this.volume === 0 ? '🔇' : '🔊'}
        </button>
        <input
          type="range"
          min="0" max="1" step="0.01"
          .value=${String(this.muted ? 0 : this.volume)}
          @input=${this.#onVolumeInput}
          style="width:60px"
        />
        <!-- Read-only quality/track display -->
        ${this.#renderTrackInfo()}
      </div>
    </div>

    ${this.debug ? html`…debug overlay…` : nothing}
  `;
}
```

**Private helpers in `VidelPresentation`:**

```ts
#onClickZone = (): void => {
  this.dispatchEvent(new CustomEvent('videl:ui:play-pause', { bubbles: true, composed: true }));
};

#onPlayPause = (): void => {
  this.dispatchEvent(new CustomEvent('videl:ui:play-pause', { bubbles: true, composed: true }));
};

#onSeekInput = (e: Event): void => {
  const frac = Number((e.target as HTMLInputElement).value);
  const dur  = this.mediaPresentationDuration ?? this.duration ?? 0;
  if (dur > 0) {
    this.dispatchEvent(new CustomEvent('videl:ui:seek', {
      bubbles: true, composed: true,
      detail: { time: frac * dur },
    }));
  }
};

#onVolumeInput = (e: Event): void => {
  const vol = Number((e.target as HTMLInputElement).value);
  this.dispatchEvent(new CustomEvent('videl:ui:volume', {
    bubbles: true, composed: true,
    detail: { volume: vol },
  }));
};

#onToggleMute = (): void => {
  // muted is independent of volume on HTMLVideoElement — dispatch a dedicated event
  this.dispatchEvent(new CustomEvent('videl:ui:mute-toggle', {
    bubbles: true, composed: true,
  }));
};

#formatTime(s: number): string {
  if (!isFinite(s)) return '--:--';
  const m = Math.floor(s / 60);
  const sec = Math.floor(s % 60).toString().padStart(2, '0');
  return `${m}:${sec}`;
}

#renderTrackInfo() {
  // Read-only display of active audio language and video quality.
  // No user interaction (ABR pinning is out of scope for this branch).
  const audioAds = [...this.querySelectorAll('videl-adaptation-set[content-type="audio"]')];
  const activeAudio = audioAds.find(a => a.getAttribute('videl-state') === 'active');
  const lang = activeAudio?.getAttribute('lang') ?? '';

  const activeRep = this.querySelector(
    'videl-representation[videl-state="active"]'
  );
  const bw = activeRep ? Number((activeRep as any).bandwidth ?? 0) : 0;
  const quality = bw > 0 ? `${Math.round(bw / 1000)}k` : '';

  return html`
    ${lang   ? html`<span class="track-info">${lang}</span>`    : nothing}
    ${quality ? html`<span class="track-info">${quality}</span>` : nothing}
  `;
}
```

## Resolved Questions

1. **Default card visibility**: `display: block` by default — visible. Can be overridden later with CSS.

2. **Single-stream `src` mode**: Mark the injected presentation with a `generated` attribute. No behavior change now, but consumers can target `videl-presentation[generated]` for future styling.

3. **`videl:ui:*` events are internal**: Fired by the presentation's shadow DOM, caught by the player. Not part of the public API. The player may selectively re-emit them in a future deliverable.

4. **ABR pinning**: Out of scope for this branch. The quality selector shows current state only — read-only display, no override.

5. **Seekbar interaction model**: A full-width `<input type="range">` lives inside the controls bar (not full-frame). It is the actual seek interaction target. Period markers are absolutely positioned decorative divs overlaid on the range track — they provide visual chapter context but are not interactive. The main frame area (above the controls bar) handles click → play/pause.

6. **`media-chrome` compatibility**: Out of scope. Ignored for this branch.
